   a. In `pollAccount()`, the existing code maps search results to `MultiAccountMessageHeader` via spread: `messages.map((m) => ({ ...m, account: accountId }))`. The `searchMessages` return type is `SearchResultItem` which extends `MessageHeader` — and `MessageHeader` now has `keywords?: string[]`. However, `searchFolder` in search-service.ts does NOT currently populate `keywords` from flags.

   To populate keywords in the cache, modify the `pollAccount` method to fetch flags and extract non-system keywords. The `searchFolder` function already fetches `{ flags: true }` in `fetchAll`, but the result mapping does not include keywords. Two approaches:

   **Preferred approach:** Modify `searchFolder` in `search-service.ts` to include keywords in the returned `SearchResultItem`. Add to the `messages.map()` return in `searchFolder`:
   ```typescript
   keywords: [...(msg.flags ?? new Set())]
     .filter((f) => !f.startsWith('\\')),
   ```
   This extracts custom keywords (those without `\` prefix) from the flags Set. System flags like `\Seen`, `\Flagged` are excluded.

   Then add `keywords?: string[]` to `SearchResultItem` in `src/types.ts` — BUT `SearchResultItem extends MessageHeader` which already has `keywords?: string[]` from Plan 01. So no type change needed; the `searchFolder` mapping just needs to populate the field.

   b. In `query()`, add `excludeKeyword` parameter and filter:
   ```typescript
   query(
     since: string,
     account?: string,
     excludeKeyword?: string
   ): MultiAccountResult<MultiAccountMessageHeader> {
   ```

   In the filter inside the `for` loop, add keyword exclusion after the `sinceTime` check:
   ```typescript
   const filtered = entries.filter(
     (m) =>
       (new Date(m.date).getTime() || 0) > sinceTime &&
       (excludeKeyword === undefined ||
         !(m.keywords ?? []).some(
           (k) => k.toLowerCase() === excludeKeyword.toLowerCase()
         ))
   );
   ```
   Note: Case-insensitive comparison per RESEARCH.md pitfall #4 — IMAP servers may normalize keyword casing.
