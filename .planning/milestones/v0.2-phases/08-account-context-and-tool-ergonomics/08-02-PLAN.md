<objective>
Make the `folder` parameter optional in `list_messages`, defaulting to `"INBOX"` when omitted, so agents can fetch inbox messages without needing to know the folder path convention.

Purpose: Agents currently must pass `folder: "INBOX"` explicitly on every list_messages call. Making folder optional removes this unnecessary friction for the most common use case.

Output: Modified src/tools/list-messages.ts (interface + handler + inputSchema) and new test cases in tests/tools/list-messages.test.ts.
