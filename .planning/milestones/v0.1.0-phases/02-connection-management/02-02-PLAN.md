Backoff constants (locked per research):
```typescript
const BACKOFF_INITIAL_MS = 1_000;
const BACKOFF_MULTIPLIER = 2;
const BACKOFF_CAP_MS = 60_000;
const BACKOFF_MAX_ATTEMPTS = 10;

function backoffDelayMs(attempt: number): number {
  const raw = BACKOFF_INITIAL_MS * Math.pow(BACKOFF_MULTIPLIER, attempt - 1);
  return Math.min(raw, BACKOFF_CAP_MS);
}
// Sequence: 1000, 2000, 4000, 8000, 16000, 32000, 60000, 60000, 60000, 60000
```

  <name>Task 1: Implement AccountConnection state machine and reconnect loop</name>
  <files>src/connections/account-connection.ts</files>
  <behavior>
    - connect() wires 'error' and 'close' listeners on the new ImapFlow BEFORE calling client.connect()
    - connect() transitions: connecting → connected on success, connecting → reconnecting on failure
    - 'close' event: if not shutting down and not already reconnecting/failed, transitions to reconnecting and starts backoff loop
    - 'error' event: logs the error message (does not re-throw; 'close' will fire after 'error' in imapflow)
    - Reconnect loop: sleeps backoffDelayMs(attempt), constructs new ImapFlow, wires listeners, attempts connect()
    - On reconnect success: transitions reconnecting → connected
    - On reconnect failure (connect() throws): increments attempt, loops again
    - When attempt > BACKOFF_MAX_ATTEMPTS: transitions to failed, logs "permanently failed", loop exits
    - AbortController: scoped to AccountConnection instance; pass signal to sleep() calls so gracefulClose() can interrupt a sleeping backoff delay
    - gracefulClose(): sets isShuttingDown = true, aborts AbortController, calls logout() if client.usable else close(), wraps in try/catch with warn log on error
    - getStatus(): returns current discriminated union state snapshot
  </behavior>
  <action>
    Create `src/connections/account-connection.ts`.

    Export `AccountConnectionStatus` discriminated union type and `AccountConnection` class.

    Private fields:
