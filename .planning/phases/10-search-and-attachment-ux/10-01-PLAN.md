<objective>
Add optional `body` parameter to `search_messages` to enable server-side body text search (SRCH-05).

Purpose: Agents can find messages by content rather than only by headers (from, subject, date). This eliminates the need for client-side content scanning and leverages the IMAP server's native SEARCH BODY capability.
Output: `body` field on `SearchParams` and `SearchMessagesParams`; `criteria.body = body` in the criteria-building block; `body` property in tool inputSchema; `body` threaded through both call sites in `handleSearchMessages`; SRCH-05 test describe block in the existing test file.
</objective>
