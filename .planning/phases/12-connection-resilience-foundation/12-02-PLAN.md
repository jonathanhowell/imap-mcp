## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imapflow error object → classifier | imapflow's typed errors are the source of truth; classifier trusts `AuthenticationFailure` instance, `tlsFailed`, and `serverResponseCode` strictly (D-07). |
| classifier output → AccountConnection state machine (Wave 2) | The classifier's binary verdict drives the `suspended` vs `reconnecting` decision; misclassification is high-impact (Pitfall 4 / 5). |

| T-12-05 | Spoofing | Attacker crafts an error object with `tlsFailed: true` to force `suspended` | accept | Errors originate from `client.connect()` rejection inside our process; no external surface. Low risk. |
