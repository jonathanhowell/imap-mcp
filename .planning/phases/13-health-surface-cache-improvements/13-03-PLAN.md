<objective>
Replace the Poller's single global `lastPollTime: Date | null` with a per-account `lastPolledAt: Map<string, Date | null>` (CACHE-01 / D-11). Add a `getLastPolledAt(accountId)` accessor (D-12). Migrate the seed-vs-incremental decision to be per-account (D-13). Stamp `lastPolledAt` AFTER `mergeIntoCache` succeeds (RESEARCH Pitfall 2) so skipped or thrown attempts retain the prior value.

Purpose: Plan 13-04's `get_new_mail` freshness response (`last_polled_at` / `cache_age_seconds` per account) reads this Map. Without per-account tracking, agents cannot tell which account is fresh vs stale. This plan is the prerequisite for Success Criterion 2.

Scope note: `isCacheReady()` remains in this plan — Plan 13-04 removes it together with the `handleGetNewMail` gate change so the delete + caller removal land in one atomic plan. Keeping it here avoids breaking the Phase 12 callers mid-wave.

Output: Per-account poll-time tracking with a public accessor. Existing 4 private-field-injection test sites migrated to the new Map shape (RESEARCH Pitfall 4). New tests for `getLastPolledAt` semantics including the skipped-account retention guarantee.
</objective>
