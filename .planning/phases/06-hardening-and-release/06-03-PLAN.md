<interfaces>
<!-- 7 MCP tools and their parameter shapes (from src/index.ts and handler files): -->

list_accounts — no parameters. Returns array of {name, display_name, host}.

list_folders(account?: string) — Returns array of {name, delimiter, flags, messages, unseen} per folder.

list_messages(folder: string, account?: string, limit?: number, offset?: number, sort?: "newest"|"oldest", unread_only?: boolean)
  Single-account: returns MessageHeader[].
  Multi-account (no account): returns { results: MultiAccountMessageHeader[], errors?: Record<string,string> }.

read_message(account: string, uid: number, format?: "clean"|"full") — Returns full message with headers + body. format="clean" strips reply chains (default).

search_messages(account?: string, from?: string, subject?: string, since?: string, before?: string, unread?: boolean, folder?: string, max_results?: number)
  Single-account: returns SearchResultItem[].
  Multi-account: returns { results: MultiAccountSearchResultItem[], errors?: Record<string,string> }.

download_attachment(account: string, uid: number, part: string) — Returns { filename, mimeType, size, data: base64 }.

get_new_mail(since?: string) — Returns { messages: CachedHeader[], polled_at: string }. Since = ISO timestamp. Without since, returns all cached unread headers.

<!-- Config schema (from src/config.ts pattern): -->
accounts[]:
  name: string (required)
  host: string (required)
  port: number (required, must be 993)
  username: string (required)
  password: string (required, use $ENV_VAR_NAME)
  display_name?: string

polling?:
  interval_seconds: number (default 300)
</interfaces>

    **5. Provider Compatibility**
    Subsections: Gmail, Generic IMAP.
    PROMINENT WARNING for Outlook/Hotmail/Microsoft 365:
    > **Outlook/Hotmail/Microsoft 365:** Microsoft deprecated Basic Auth for IMAP in October 2022.
    > Modern Microsoft accounts require OAuth2, which this server does not currently support.
    > If you are using an Outlook, Hotmail, or Microsoft 365 account, it will not work with this server.
    Gmail: App Passwords required (Google account password won't work); enable IMAP in Gmail settings.

    **6. Tool Reference**
    One H3 heading per tool. Under each tool: a parameter table (columns: Parameter, Type, Required, Description) and a brief response shape description or example.
    Cover all 7 tools: list_accounts, list_folders, list_messages, read_message, search_messages, download_attachment, get_new_mail.
    Note the 200-result maximum for list_messages and search_messages.
    Note that multi-account responses return `{ results, errors? }` wrapper while single-account returns flat array.

    **7. Example Agent Prompts**
    3 concrete examples showing what an agent can ask:
    - "Show me all unread emails from the last 24 hours across all my accounts"
    - "Search for emails from GitHub about pull requests in my work account from this week"
    - "Read the most recent email from alice@example.com and summarize it"

    **8. Troubleshooting**
    H3 subsections:
    - "Connection refused / timeout" — check port is 993, not 143 or 587; TLS is enforced
    - "Authentication failed (Gmail)" — use an App Password (myaccount.google.com/apppasswords); enable IMAP in Gmail settings (Settings → See all settings → Forwarding and POP/IMAP)
    - "Authentication failed (generic)" — verify username/password; check if your provider requires app-specific passwords
    - "Outlook/Microsoft accounts not working" — repeat the Basic Auth deprecation note; suggest alternatives not in scope
    - "Server crashes on startup" — check config file path; check that required env vars are set
    - "Performance: slow on large mailboxes" — note that `folder='all'` in search_messages searches every folder sequentially

    **9. Contributing**
    - Dev setup: `npm install`, `npm run build`, `npm test`
    - TypeScript strict mode throughout — no `any` types without explicit justification
    - All log output goes to stderr via `logger.ts` — never use `console.log` (enforced by ESLint)
    - Pre-commit hooks run lint + tests automatically via Husky
    - Install gitleaks for secret scanning: `brew install gitleaks`
    - PR flow: branch from main, open PR, CI must pass
