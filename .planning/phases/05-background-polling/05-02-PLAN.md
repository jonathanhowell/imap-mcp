Create `src/polling/poller.ts` implementing the `Poller` class. Key design decisions (from CONTEXT.md + RESEARCH.md):

**Class structure:**
```typescript
import { ConnectionManager } from "../connections/index.js";
import { searchMessages } from "../services/search-service.js";
import { logger } from "../logger.js";
import type { MultiAccountMessageHeader, MultiAccountResult } from "../types.js";

export class Poller {
  private cache = new Map<string, MultiAccountMessageHeader[]>();
  private lastPollTime: Date | null = null;
  private stopped = false;

  constructor(
    private readonly manager: ConnectionManager,
    private readonly intervalSeconds: number = 300
  ) {}

  start(): void { ... }
  stop(): void { ... }
  isCacheReady(): boolean { ... }
  query(since: string, account?: string): MultiAccountResult<MultiAccountMessageHeader> { ... }
  private async runLoop(): Promise<void> { ... }
  private async poll(): Promise<void> { ... }
  private async pollAccount(accountId: string): Promise<void> { ... }
  private mergeIntoCache(accountId: string, incoming: MultiAccountMessageHeader[]): void { ... }
}
```

**Critical implementation rules:**
1. `start()` sets `this.stopped = false` then calls `void this.runLoop()`
2. `runLoop()` calls `await this.poll()` then, if not stopped, schedules next with `globalThis.setTimeout(() => void this.runLoop(), this.intervalSeconds * 1000)`
3. `stop()` sets `this.stopped = true` — no further setTimeout after the current poll completes
4. `poll()` iterates `this.manager.getAccountIds()` sequentially (NOT Promise.allSettled — avoid concurrent IMAP lock contention). Per-account: try/catch around `pollAccount`, log errors with `logger.error`, continue.
5. After all accounts polled, set `this.lastPollTime = new Date()`
6. `pollAccount(accountId)`: get client via `getClient()`, guard with `'error' in result`. Call `searchMessages` with:
   - Startup (lastPollTime === null): `since = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000).toISOString()`, `maxResults: 1000`
   - Incremental: `since = new Date(this.lastPollTime.getTime() - 24 * 60 * 60 * 1000).toISOString()`, `maxResults: 100`
   - Always: `folder: "INBOX"` (per CONTEXT.md — poll INBOX only)
   - Build `MultiAccountMessageHeader[]` by spreading each result with `{ account: accountId }`
   - Call `this.mergeIntoCache(accountId, headers)`
7. `mergeIntoCache`: get existing array (or []), build UID Set from existing, append only entries with uid not in set
8. `query(since, account?)`:
   - `sinceTime = new Date(since).getTime() || 0`
   - If account provided: iterate [account]; else iterate all accountIds
   - For each id: get cache entry; if missing add to errors; else filter by `new Date(m.date).getTime() > sinceTime`
   - Sort results newest-first: `(a, b) => (new Date(b.date).getTime() || 0) - (new Date(a.date).getTime() || 0)`
   - Return `{ results, ...(errors has keys ? { errors } : {}) }`
   - NOTE: `query()` does NOT check `isCacheReady()` — that check belongs in the handler layer
